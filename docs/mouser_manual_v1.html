Section 01

Home Screen

When you launch Mouser, the Home Screen offers four entry points. The first two are also accessible via File in the menu bar.

✨
New Experiment
Create a new experiment file and save it to a folder of your choice.
📂
Open Experiment

🔬
Equipment Testing
Validate connected devices before collecting real experimental data.
🔌
Serial Port Testing
Configure and verify serial port connections for readers and devices.

Section 02

Create a New Experiment

Go to Home → New Experiment (or File → New Experiment) and complete the three-step setup wizard.

Step 1 — Fill in experiment details
FieldRequired?Description
Experiment NameRequiredA clear, descriptive name for your experiment.
PasswordOptionalSetting a password saves the file as an encrypted .pmouser.
InvestigatorsOptionalNames of personnel running the experiment.
SpeciesOptionalAnimal species being studied.
Measurement ItemsOptionalWhat you plan to record — e.g. weight, temperature.
Animal / Group CountOptionalNumber of animals, groups, and animals per cage.
Uses RFIDOptionalEnable if animals will be identified by RFID tag scans.
Step 2 — Configure groups & choose input method

Name each group, then select how measurements will be entered:

⚡ Automatic
For workflows using connected hardware or RFID scanning. Measurements are captured directly from the device.
⌨️ Manual
For workflows where the investigator types measurement values directly into Mouser.
Step 3 — Review & save

Review the Summary page, click Create, then choose a folder to save the experiment file.

⚠️
Avoid saving experiment files to network drives or removable storage — these can be unreliable and may cause data loss.
📂
Section 03

Open an Existing Experiment

Go to Home → Open Experiment (or File → Open Experiment) and select your file. Mouser supports two formats:

📄 .mouser
Standard experiment database file. Opens directly — no password required.
🔒 .pmouser
Password-protected encrypted file. You will be prompted to enter the correct password before the experiment opens.

After opening, Mouser takes you directly to the Experiment Menu for that file.

🔬
Section 04

Equipment Testing

Use Home → Equipment Testing before live sessions to verify connected measurement devices are operating correctly. Do not skip this step when using new or reconfigured hardware.

1
Connect devices
Attach your scale, sensor, or other supported measurement hardware before opening the Equipment Testing view.
2
Run signal checks
Confirm that values update in Mouser when the device state changes — for example, placing a weight on a scale or triggering a sensor.
3
Resolve warnings
Address any unrecognized or unstable devices shown in the test view before proceeding to Data Collection.
🚨
Do not begin Data Collection until all required devices are recognized and returning stable readings in the Equipment Testing view.
🔌
Section 05

Serial Port Testing

Use Home → Serial Port Testing to confirm your RFID reader or serial-connected device is reachable on the expected COM port before starting an experiment session.

CheckWhat To Confirm
Port SelectionThe selected COM port matches the device shown by your operating system.
Communication SettingsBaud rate and serial parameters match your device specification.
Live Read TestIncoming data appears in Mouser when scanning a tag or sending a test value from the device.
⚠️
If no data appears during the live read test, close any other software that may be holding the same COM port open, then retest.
🗂️
Section 06

Experiment Menu

The Experiment Menu is your central hub once an experiment is open. All major actions are accessible from here.

Menu ItemPurpose
📡 Map RFIDLink animals to RFID tags before data collection. Required if RFID is enabled.
📋 Data CollectionRecord today's measurements for each animal in the experiment.
📊 Data AnalysisView trends and export data to spreadsheets or statistics tools.
🔍 Review SummaryConfirm core experiment settings before collecting or exporting data.
👥 InvestigatorsView or update the list of personnel on this experiment.
📓 Experiment NotebookRecord narrative context, protocol deviations, and session notes.
⚙️ Group ConfigurationOptionalModify group names and input method after experiment creation.
🏠 Cage ConfigurationOptionalModify cage assignments — move, swap, autosort, or randomize animals.
🗑️ Delete ExperimentDestructivePermanently removes the experiment file. Cannot be undone.
🚨
Delete Experiment permanently deletes the file — there is no recycle bin or undo. Use with extreme caution.
📓 Experiment Notebook Accessible from Experiment Menu
Use the Experiment Notebook to record narrative context that does not belong in numeric measurement fields — protocol deviations, environment observations, and session-to-session notes for investigators.
✓ Use Notebook For
Protocol deviations, environment notes, and event timelines.
Session-to-session reminders for investigators.
Context for interpreting unusual outliers during analysis.
✗ Avoid Using Notebook For
Primary measurement values that belong in Data Collection fields.
Temporary scratch notes you do not intend to preserve.
Replacing configured metadata such as group definitions or investigator list.
💡
Keep entries brief, date-stamped, and specific so they remain useful during later reviews and exports.

Section 07

Map RFID

If your experiment uses RFID, you must link each animal to its tag before starting data collection. Open Map RFID from the Experiment Menu.

1
Open Map RFID
Select Map RFID from the Experiment Menu to open the mapping interface.
2
Start scanning
Begin with your serial RFID reader or HID keyboard-wedge reader, depending on your hardware setup.
3
Map each animal
For each animal: select or confirm the target animal ID, scan its RFID tag, then verify the mapping appears in the table.
4
Save & return
Once all animals are mapped, save and return to the Experiment Menu to begin data collection.
⚠️
Mouser will block or warn when starting Data Collection if any expected animals remain unmapped. Complete all RFID mappings first.
📋
Section 08

Data Collection

Use Data Collection to record measurements for today's session. Open it from the Experiment Menu.

📡 RFID Experiments
Click Start scanning, then scan an animal's tag to pull it up automatically.
⌨️ Non-RFID Experiments
Click Start to step through animals in sequence using the auto-increment flow.
Animal status progression
⏳ Pending → 🔄 In Progress → ✅ Done

The status column updates in real-time. Continue until every animal shows Done.

💾 Saving behaviour
File TypeHow saving works
📄 .mouserData commits as you collect. Mouser can also auto-save the open copy back to the original file during the session.
🔒 .pmouserSaving is handled through the password-protected save flow at session end.
📤
The Data Collection page includes an Export CSV button for a quick export of all current experiment data.

Section 10

Group Configuration

Use Group Configuration from the Experiment Menu when group names or the input workflow must be adjusted after experiment creation. It is not part of the core workflow — open it only when adjustments are needed.

1
Open Group Configuration
From the Experiment Menu, click Group Configuration to open the settings panel.
2
Rename or review groups
Update group labels to match your active protocol naming conventions.
3
Confirm input mode
Verify whether this experiment should use Automatic or Manual entry going forward, and update if needed.
💡
Group Configuration can be revisited at any point during the experiment — changes take effect immediately.

Section 11

Cage Configuration

Cage Configuration lets you modify how animals are distributed across cages and groups. Open it from the Experiment Menu whenever reassignment is needed.

Quick actions available
🔀
AutoSort
Automatically distribute animals into cages based on defined criteria.
🎲
Randomize
Randomly assign animals to cages to reduce selection bias.
↔️
Swap
Swap two animals between their respective cages.
↗️
Move Selected
Move a selected animal to a different cage or group.
📡
If RFID is enabled, you can scan an animal's tag to select it quickly in Cage Configuration, depending on your device and input mode.

ActionWhy It Matters

Edit investigator listKeeps records current when staff rotate or responsibilities change mid-experiment.

Section 13

Review Summary

Open Review Summary from the Experiment Menu to quickly confirm core experiment settings before collecting or exporting data.

Recommended Pre-Session Check
1
Confirm experiment name, species, and measurement items are correct and match the current protocol.
2
Verify group setup and animal counts match the current session plan before starting Data Collection.
3
Check RFID usage and mapping status — ensure all animals are mapped if RFID is enabled.
✅
Running through the pre-session check before each collection session helps catch misconfiguration before it affects your data.

Section 14

Troubleshooting

Quick checks for the most common issues encountered in Mouser.

IssueRFID tags are not being read
Go to Home → Serial Port Testing and confirm the reader is detected and correctly configured.
If using a HID keyboard-wedge reader, verify the system is receiving keystrokes from the device — test in a plain text editor first.
IssueCannot start data collection
If RFID is enabled, open Map RFID and complete the mapping for all animals before proceeding to data collection.
IssueDevice not recognized in Equipment Testing
Ensure the device is physically connected and powered on before opening the Equipment Testing view.
Check that no other software is holding the COM port. Close competing applications and reopen Equipment Testing.
IssueData does not appear to be saving
Confirm you have a valid experiment file open — data will not persist from a temporary or scratch state.
For .pmouser files, ensure you used the correct password and the file was closed normally at the end of the last session.
